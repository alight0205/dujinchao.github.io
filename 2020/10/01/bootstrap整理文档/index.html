引入方式

  1. 下载本地引入
    • 3.x 版本
      • Bootstrap：编译压缩版（有字体文件夹）
      • Source code：源文件
      • Sass：…
    • 4.x 版本 - Complied CSS and JS：编译过后的 css 和 js（没有字体文件夹） - Source files：源文件
  2. 通过 CDN 引入：通过 cdn 引入文件 cdn（内容资源分发网络，将资源分发离你近的服务器，让你迅速获取）
  • 引入方式详见：https://getbootstrap.com
  • link 标签中的陌生属性：
    • integrity：里面是一段加密的哈希值
    • crossorigin：发送一个跨域的请求
    • 当我们去引入的时候，他需要去带上哈希值去进行请求，服务器收到之后会进行匹配，如果哈希值正确，可以用，不正确不给用，主要是为了安全。
    • meta 标签 viewport，就是为了做移动端的适配
    • meta 标签中 shrink-to-fit=no，给 ios9 做了一个适配，反正 copy 官网给的 mata 就对了

响应式原理

4.x 的 4 个核心部分

  • Layout：栅格系统
  • Content：内容，一些排版的设置
  • Components：组件，把一些小的功能都做成了一个个的组件
  • Utilities：工具部分，像浮动、阴影、颜色…
  • Extend：扩展，插件、字体图标(非核心)

响应式原理

响应式所具有的特点：

  1. 网页的宽度自动调整
  2. 尽量少用绝对宽度
  3. 字体要使用 rem、em 做为单位
  4. 布局要使用浮动、弹性布局

@规则

- @chartset 定义编码
- @import 引入css文件（css模块化）
- @font-face 自定义字体
- @keyframes animation里的关键帧
- @media 媒体查询

媒体查询

根据一个或多个基于设备类型、具体特点和环境来应用样式 -> @media

使用方法

  • 官方参考：https://drafts.csswg.org/mediaqueries/

媒体类型（指的就是设备的情况）

  • all 所有设备
  • print 打印机设备
  • screen 彩色的电脑屏幕
  • speech 听觉设备（针对有视力障碍的人士，可以把页面的内容以语音的方式呈现的设备）
  • 注意：tty、tv、projection、handheld、braille、embossed、aural 等几种类型在媒体查询 4 中已经废弃

媒体特性（设备当中的一些属性，宽高..)

  • width 宽度
    • min-width 最小宽度
    • max=width 最大宽度
    • 使用 @media(max-width:400px){}
  • orientation 方向
    • landscape 横屏（宽度大于高度）
    • portrait 竖屏（高度大于宽度）
    • 使用 @media(orientation:landscape){}
  • aspect-ratio 宽高比
  • -webkit-device-pixel-ratio 像素比(webkit 内核私有属性)

逻辑运算符

  • and 合并多个媒体查询（&&）
  • , 匹配某个媒体查询（||）
  • not 对媒体查询结果取反
  • only 仅在媒体查询匹配成功后应用样式（防范老旧浏览器）
    • @media only (min-width:500px){…} 老旧浏览器不支持媒体查询，会直接解析内部的样式，加 only 可防止这种情况

Layout：布局

栅格系统：

bootstrap 中，它把屏幕分成了十二列，默认提供了 5 个默认响应层。

3.x 和 4.x 的区别

  1. 响应层分配数量不同。3.x 分了 4 和，4.x 分了 5 个
  2. 响应层的分区像素不一样，详情看文档。
  3. 4.x 布局使用的是弹性盒模型，3.x 是用的布局是浮动

栅格系统的使用

响应式的分层简写

  • .col- 超小屏，屏幕尺寸<576px，container 的尺寸为 None(auto)
  • .col-sm- 小屏，屏幕尺寸>=576px， container 的尺寸为 540px
  • .col-md- 中等屏，屏幕尺寸>=768px， container 的尺寸为 720px
  • .col-lg- 大屏，屏幕尺寸>=992px， container 的尺寸为 960px
  • .col-xl- 超大屏，屏幕尺寸>=1200px， container 的尺寸为 1140px

.container & .container-fluid

所有栅格系统，页面中所有的内容都要放在这两个 div 当中

  • .container-fulid 宽度是 100%，为视口宽度
  • .container 宽度是一个具体的数值，会根据屏幕尺寸进行变化（如上图）
  • .row 行（相当于 tr）
  • .col- 列（1 行 12 列）
    • .col 等宽列，剩余空间等宽分布。
      • .w-100 多行等宽列，这个代表横跨 100%。可以给等宽列换行
    • .col-xl-number number 表示所占的列。
    • .col-xl-1 如果屏幕尺寸< 1200 的时候，所有列宽度都会变为 100%
    • .col-number 所有尺寸下都占用相同列数。
    • .col-{breakpoint}-auto 根据内容调整列的宽度

混合排列或组合模式

  2. 在大屏幕下一行显示 4 个 div，一个 div 占 3 列

  • ==垂直对齐==
  1. 行的对齐方式
    • align-items-start 顶部对齐

  5. justify-content-between 左右两端对齐（元素之间的间距是自动平分的）
  • ==列排序==

重置

将一些标签原有的样式去掉，然后在重新给标签定义我们需要的属性，便于我们来使用。

重置 css 中选择覆盖和重定义的元素方针以及理由：

  • 重置浏览器默认值，使用 rem 作为尺寸规格单位，代替 em 用于 指定可缩放的组件的间隔与缝隙。
  • 大化避免使用 margin-top ,防止使用它造成的垂直外排版（边 距）混乱所造成之意想不到结果。更重要的是，一个单一方向的 margin 是一个简单的构思模型。
  • 为了易于跨设备缩放， block 块元素必须使用 rem 作为 margin 的单位。
  • 保持 font 相关属性小的声明，尽可能地使用 inherit 属性， 不影响容器溢出。inherit：可以继承

默认效果

  • 全局性地将每一个元素的 box-sizing 属性(包括 _:before 、 _:after 都设置为 border-box -以确保 DIV 元素自身定义的宽度 不会因为 border 或 padding 而超过。
  • 根元素没有声明 font-size 属性，但被假定为 16px 大 小（这是目前Chrome等浏览器默认值)，然后在此基础上采用font-size:1rem 的比例应用于 上，使媒体查询能够轻松的实现缩放，大程序保障用户偏好和易于访问。
  • 元素被赋予一个全局性的 font-family 和 lineheight ，其下面的诸多表单元素也继承此属性，以防止字体大小 错位冲突。
  • 为了安全起见， 的 background-color 的默认值赋为 #fff 。

文字字体属性

参考文档

标题和段落

所有标题和段落元素(如说 <h1> 以及 <p> 都被重置，系统移除它们的上外边距 margin-top ，标题添加外边距为 margin-bottom:.5rem ，段落 元素 <p> 添加了外边距 margin-bottom:1rem 以形成简洁行距

列表

  • 移除所有的列表元素(<ul>、<ol>、 and <dl>）的外边距 margin-top，并设置为 margin-bottom: 1rem，被嵌套的子列表没 margin-bottom 值。
  • dl/dt/dd 适用场景：dt 为名词，dd 为名词的解释。
    • 为了得到更简单的样式、清晰的等级以及更好的间距，描述列表<dd>有一个更优级别的 margin 属性定义；其 margin-left 被重置为 0、并添加 margin-bottom: .5rem 值；另一方面
      的字体是粗体

pre 预先格式化文本

<pre> 预格式化标签 pre 标签可定义预格式化的文本。被包围在<pre>标签元素中的文本通常会保留空格和换行符。而文本也会呈现为等宽字体。

bootstrap 重置了 pre 元素，移除了它的 margin-top 属性并用 rem 作为 margin-bottom 的单位。
